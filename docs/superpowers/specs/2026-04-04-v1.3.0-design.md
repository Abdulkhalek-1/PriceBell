# PriceBell v1.3.0 Design Spec

**Date:** 2026-04-04  
**Status:** Approved  
**Branch:** v1.3.0  

---

## Overview

v1.3.0 is a quality and distribution release. It ships bug fixes, memory safety improvements, proper OS installer packages, a Notepad++-style auto-updater, improved notifications, bundled fonts, publisher signing metadata, and a native plugin starter example.

Price history was explicitly deferred to a future release.

## track-a — Bug Fixes & Startup Behavior

### Fullscreen/Minimize Bug

**Problem:** When the window is in fullscreen and minimized, restoring it produces a broken UI state.

**Fix:** In `MainWindow`, override `changeEvent(QEvent*)`. On `QEvent::WindowStateChange`, persist the current `windowState()` to `QSettings`. On `showEvent` / restore from tray, read the saved state and call `showFullScreen()` if it was fullscreen, otherwise `showNormal()`. Never call `show()` blindly on restore.

### Memory Leaks After Price Checks

**Problem:** `QNetworkReply*` objects created during price checks are not consistently deleted after use.

**Fix:** Audit all handlers (`SteamHandler`, `UdemyHandler`, `AmazonHandler`, `GenericWebHandler`) and `HttpClient`. Every `QNetworkReply*` must call `reply->deleteLater()` in its finished/error lambda or slot. Add AddressSanitizer (`-fsanitize=address`) to the CMake `Debug` build type and run it in CI on the test suite to catch remaining leaks.

### Launch Minimized to Tray

**Problem:** App always opens a visible window on startup even when the user wants it in the background.

**Fix:**

- Add `--minimized` CLI argument parsed in `main.cpp`.
- Add a **"Launch minimized to tray"** toggle in `SettingsDialog` (General tab), stored in `QSettings` under `startup/launchMinimized`.
- On startup: if either the CLI flag is set or the setting is enabled, skip `mainWindow.show()` and call `trayIcon->show()` directly. The window appears only when the user clicks the tray icon.

## track-b — Fonts & Signing

### Bundle Noto Sans Fonts

**Problem:** On systems without Noto Sans installed, Qt falls back to Arial (or worse), breaking visual consistency and RTL Arabic rendering.

**Fix:**

- Download and add to `assets/fonts/`:
  - `NotoSans-Regular.ttf`
  - `NotoSans-Bold.ttf`
  - `NotoNaskhArabic-Regular.ttf` (covers Arabic RTL i18n)
- Register all three in `assets/resources.qrc` under a `fonts/` prefix.
- In `main.cpp`, call `QFontDatabase::addApplicationFont(":/fonts/NotoSans-Regular.ttf")` (and the others) before creating `QApplication`.
- Set the app-wide default font: `app.setFont(QFont("Noto Sans", 10))`.

### Publisher Metadata (Version Info)

**Problem:** The binary has no publisher identity visible in OS file properties.

**Fix:**

- Add `cmake/VersionInfo.cmake` included from `CMakeLists.txt`.
- **Windows:** Generate `src/version.rc` with `VERSIONINFO` resource block. Fields: `FileDescription`, `ProductName` ("PriceBell"), `CompanyName`, `LegalCopyright`, `FileVersion`, `ProductVersion`, `InternalName`, `OriginalFilename`. Add `src/version.rc` to the executable sources on Windows.
- **macOS:** Generate `Info.plist` with `CFBundleName`, `CFBundleVersion`, `CFBundleShortVersionString`, `NSHumanReadableCopyright`, `CFBundleIdentifier` (`com.abdulkhalek.pricebell`).

### Self-Signed Certificate Signing

**Fix:** Extend `scripts/sign.sh`:

- Generate a self-signed X.509 cert with `CN=Abdulkhalek Muhammad, O=PriceBell` using OpenSSL if not already present.
- **Windows:** Sign the `.exe` with `signtool sign` using the generated `.pfx`.
- **Linux:** Detach-sign the binary with `gpg --detach-sign`, producing a `.sig` file alongside the release artifact.
- **macOS:** Sign the `.app` bundle with `codesign --sign` using the self-signed identity.
- Note: Self-signed certs show "Unknown publisher" on Windows SmartScreen — this is expected behavior.

## track-c — Notifications

### Deep Link: Click Alert → Open Product URL

**Problem:** Clicking a price alert notification does nothing useful.

**Fix:** In `AlertManager::fireAlert()`, connect `QSystemTrayIcon::messageClicked` signal to a lambda that calls `QDesktopServices::openUrl(QUrl(product.url))`. Since `messageClicked` is a single-shot signal per message, disconnect the previous connection before connecting the new one to avoid stale URL opens.

### Update Available Tray Notification

**Problem:** When `UpdateChecker` finds a new version, there is no tray notification — only a dialog.

**Fix:** In `UpdateChecker`, after confirming a newer version is available, fire a tray notification: `trayIcon->showMessage("PriceBell Update Available", "Version X.Y.Z is ready. Click to see what's new.", QSystemTrayIcon::Information, 8000)`. The `messageClicked` signal opens the GitHub releases page. The full update dialog (`UpdateDialog`, see track-d) is shown separately.

## track-d — Updater (Notepad++ Model)

**Depends on:** track-c (tray notification infrastructure)

### UpdateDialog

A modal dialog shown when a new version is detected (triggered by startup check or tray notification click):

- Displays: current version, new version, brief changelog summary (fetched from GitHub Release body, truncated to ~300 chars).
- Buttons: **"Update Now"**, **"Later"**, **"Skip This Version"** (stores skipped version in `QSettings`).

### UpdateDownloader

New class `include/utils/UpdateDownloader.h`:

- `UpdateChecker` already fetches the GitHub Releases API JSON. `UpdateDownloader` receives the full assets array and selects the correct asset by matching the current platform: `.deb` on Linux, `.exe` on Windows, `.dmg` on macOS (detected via `Q_OS_LINUX` / `Q_OS_WIN` / `Q_OS_MACOS` compile guards).
- Downloads to a temp file (`QStandardPaths::TempLocation`).
- Emits `progress(int percent)` and `finished(QString tempFilePath)` signals.
- `UpdateDialog` shows a `QProgressBar` during download, replaces buttons with a cancel option.

### Install & Relaunch

On download complete:

1. `QProcess::startDetached(tempFilePath)` — launches the installer.
2. `QApplication::quit()` — exits PriceBell so the installer can replace the binary.

## track-e — Installer Packaging

### Linux — DEB & RPM (CPack)

In `CMakeLists.txt`, add CPack configuration:

- `CPACK_GENERATOR`: `DEB;RPM`
- `CPACK_DEBIAN_PACKAGE_MAINTAINER`, `CPACK_DEBIAN_PACKAGE_DEPENDS` (Qt5 runtime libs)
- `CPACK_RPM_PACKAGE_REQUIRES`
- Post-install script: installs `pricebell.desktop` to `/usr/share/applications/`, registers icon at `/usr/share/icons/hicolor/`.

### Windows — NSIS Installer

`packaging/windows/installer.nsi`:

- Installs to `$PROGRAMFILES\PriceBell\`.
- Creates Start Menu shortcut and optional Desktop shortcut.
- Registers uninstaller in `HKLM\Software\Microsoft\Windows\CurrentVersion\Uninstall`.
- Includes Visual C++ runtime check.

### macOS — DMG (CPack)

- `CPACK_GENERATOR`: `DragNDrop`
- Bundles the `.app` with a background image and symlink to `/Applications`.

### CI Integration

Add a `package` job to `.github/workflows/release.yml`:

- Runs after the build job on each platform.
- Calls `cpack` to generate the installer artifact.
- Uploads artifact to the GitHub Release with `gh release upload`.

## track-f — Native Plugin Starter Example

New directory: `examples/plugin-starter/`

Contents:

- `CMakeLists.txt` — standalone project linking against PriceBell's `IPlugin2.h` (copied or referenced via install path).
- `ExamplePlugin.h` / `ExamplePlugin.cpp` — minimal implementation of `IPlugin2`: returns a fixed price of `9.99`, implements `settingsWidget()` returning `nullptr`.
- `example-plugin.json` — Qt plugin metadata file.
- `README.md` — step-by-step build instructions, explains each required method, links to `docs/PLUGINS.md`.

**CMake opt-in:** `option(BUILD_EXAMPLES "Build example plugins" OFF)`. The examples directory is only added via `add_subdirectory(examples/plugin-starter)` when the option is ON. CI does not build examples by default.
