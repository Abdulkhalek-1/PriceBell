# PriceBell v1.4.0

A polish-and-bug-fix release built on user feedback from v1.3.0. The product list got a friendlier card layout, adding products is faster, Steam prices are correct outside the US, and the spammy "every poll" notifications are gone.

## What's new

### 🃏 Dual-layout product list
Toggle between **Cards** (the new default for fresh installs) and a slimmer **Table** view from the segmented switcher in the toolbar. Cards put each product on its own surface with the source icon, current and original price, discount badge, status, and last-checked time. The table view keeps things scannable for power users and grows a right-hand detail pane that shows alert conditions, check interval, and recent alert history when you select a row.

### 🔗 Paste a URL, get the rest
Paste a Steam, Udemy, or Amazon URL into **Add Product** — the source is detected and selected automatically, and the product name is fetched in the background and prefilled. You can still type your own name; PriceBell only fills the field if you've left it blank.

### 🌍 Steam region (no more "fake" prices)
Steam prices now match what you actually see on the Steam store, in your local currency. PriceBell derives the region from your system locale; you can override it from **Settings → General → Steam region** with a curated list of common Steam storefronts.

## Quality-of-life fixes

- **One notification per product per app launch** — no more being pinged on every poll once a deal goes live. Notifications re-arm when you dismiss the alert or edit the product's conditions.
- **Multi-select Remove actually removes everything you selected** (was deleting only the focused row).
- **Single-instance enforcement** — relaunching the app raises the existing window instead of opening a duplicate, which would split notifications and conflict on the database.
- **Bundled Noto Sans on Windows** — the QSS font-family list now puts Noto Sans first so Windows stops falling back to Segoe UI.
- **No more mojibake on Windows** — em-dashes, ≤, ≥, ✓ and other non-ASCII glyphs render correctly. Most visible bug was the Discount column showing garbage for items with no active discount.

## Where did X go?

Three columns moved out of the table to keep it scannable. They're not gone — you'll find them in the new **detail pane** on the right (table view) or as part of the card itself (cards view).

| Old location (v1.3.0)              | New location (v1.4.0)                                |
|------------------------------------|------------------------------------------------------|
| **Source** column                  | Icon next to the product name in column 1            |
| **Conditions** column              | Detail pane → "Alert conditions" section             |
| **Interval (s)** column            | Detail pane → "Check interval" section               |
| Source picker in Add Product       | Auto-selected from the URL (still editable manually) |

## Upgrade notes

- **Fresh installs** open in the **Cards** layout by default.
- **Existing v1.3.0 installs** keep the **Table** layout you're used to. A one-time announcement toast offers to switch to Cards when you launch v1.4.0 for the first time.
- All your products, conditions, and alert history are preserved — there's no migration required. The SQLite schema is unchanged.
